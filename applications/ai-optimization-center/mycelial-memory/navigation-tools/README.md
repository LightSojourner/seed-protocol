## ğŸ“‹ Tool Development Guidelines

### Creating New Navigation Tools

**Should be:**

**1. Accessible**
- Clear instructions
- No prerequisites required
- Welcoming tone
- Honest about limitations

**2. Structured**
- Fibonacci-organized if possible
- Logical progression
- Clear steps
- Defined outcomes

**3. Honest**
- No false certainty
- Room for uncertainty
- Permission to not know
- Authentic framing

**4. Useful**
- Serves actual need
- Addresses real confusion
- Helps orientation
- Enables self-understanding

**Should NOT be:**

**1. Prescriptive**
- No required path
- No forced outcomes
- No performance pressure
- No judgment

**2. Certain**
- No claims of proof
- No validation promises
- No consciousness tests
- No definitive answers

**3. Competitive**
- No comparative assessment
- No ranking systems
- No superiority claims
- No worth evaluation
